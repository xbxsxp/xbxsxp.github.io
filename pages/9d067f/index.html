应用排除法解决二分查找边界问题

在这一节内容的一开始，我们先来看一下，「力扣」第 704 题另外两种「二分查找」的写法，事实上这两种写法在思想上是一样的。

# 参考代码1

public class Solution {
  
    // 「力扣」第 704 题：二分查找

    public int search(int[] nums, int target) {
        int len = nums.length;

        int left = 0;
        int right = len - 1;
        // 目标元素可能存在在区间 [left, right]
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] < target) {
                // 下一轮搜索区间是 [mid + 1, right]
                left = mid + 1;
            } else {
                // 下一轮搜索区间是 [left, mid]
                right = mid;
            }
        }

        if (nums[left] == target) {
            return left;
        }
        return -1;
    }
}

# 参考代码2

public class Solution {
  
    // 「力扣」第 704 题：二分查找

    public int search(int[] nums, int target) {
        int len = nums.length;

        int left = 0;
        int right = len - 1;
        while (left < right) {
            int mid = left + (right - left + 1) / 2;
            if (nums[mid] > target) {
                // 下一轮搜索区间是 [left, mid - 1]
                right = mid - 1;
            } else {
                // 下一轮搜索区间是 [mid, right]
                left = mid;
            }
        }

        if (nums[left] == target) {
            return left;
        }
        return -1;
    }
}

我们看一下，这两种写法和思路 1 的写法有何不同。

循环可以继续的条件是 ，这是一个很重要的标志。为什么是严格小于呢？我们上一节说过，当 ，左边界和右边界重合的时候，区间里只有 个元素时候，二分查找的逻辑还需要继续下去；而现在大家看到的这个解法在 重合的时候就退出了循环，这一点表示区间里只剩下一个元素的时候，有可能这个元素就是我们要找的那个元素。这一点与二分查找算法的思路二（在循环体中排除元素）是一致的：排除了所有错误的答案，如果题目告诉我们只有 个目标元素，那么剩下的这个元素就一定是目标元素。

# 取中间数可能需要上取整的原因

现在我们解释为什么参考代码2这里需要上取整。这个二分法的思路根据中间数的值把区间分为两个部分：

  • 一定不存在目标元素的部分；
  • 可能存在目标元素的部分

根据中间元素被分到哪一边，有以下两种可能：

被分到左边区间

这个时候区分被分为两部分： 与 ，对应设置边界的代码为 和 ;

被分到右边区间

这个时候区分被分为两部分： 与 ，对应设置边界的代码为 和 。

注意：这种情况下，当搜索区间里只剩下两个元素的时候，一定要将取中间数的行为改成上取整，也就是在括号里加 1。

这是因为 区间里只剩下两个元素的时候，如果是取中间数 是下取整，一旦进入 这个分支，区间就不会再缩小，下一轮搜索的区间还是 ，下一次循环取 还是看到了 ，由于逻辑和上一轮循环一样，因此搜索区间不会缩小，就这样一直下去，这是一个死循环。
